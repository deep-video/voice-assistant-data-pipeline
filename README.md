## 项目背景

### 核心挑战

1. **缺乏真实数据**：项目初期没有真实用户语音数据
2. **数据真实性差**：早期生成的数据过于简单、模板化
3. **意图边界模糊**：相似意图之间容易混淆（如音乐推荐 vs 媒体推荐）
4. **多样性不足**：句式单一，过度依赖"推荐"、"来点"等固定模式

### 目标

生成 1000+ 条/意图 的高质量训练数据，覆盖 13 个意图类别，包含：
- 种子数据（20%）
- 扩展数据（70%）
- 边界样本（10%）

## 架构演进

### 第一阶段：规则驱动（已废弃）

**初始思路**：基于规则模板 + 随机槽位填充

```python
# 伪代码示例
templates = ["推荐{type}音乐", "来点{mood}的歌", "播放{artist}的歌曲"]
slots = {"type": ["流行", "古典", "摇滚"], "mood": ["开心", "伤感", "轻松"]}
```

**问题**：
- ❌ 规则有限，无法覆盖真实场景的多样性
- ❌ 随机组合的句子不通顺
- ❌ 过度依赖硬编码的关键词
- ❌ 缺乏真实用户说话的口语化特征

**结论**：规则方法无法生成自然、多样的训练数据

---

### 第二阶段：LLM 驱动的种子 - 扩展架构

#### 核心设计

**两阶段生成**：
1. **种子生成**：LLM 一次性生成少量高质量种子（10-100 条）
2. **种子扩展**：基于每个种子生成多个变体（8-10 倍扩展）

**为什么这样设计？**

| 设计决策 | 原因 |
|---------|------|
| 种子阶段少量生成 | LLM 一次性生成的质量高于分批生成，便于质量控制 |
| 扩展阶段大批量 | 基于种子变体可快速扩大数据规模 |
| 质量过滤环节 | 剔除不符合意图定义的低质量样本 |

#### 种子生成策略演进

**尝试 1：单次调用生成 100 条**
```python
# 一次性让模型生成 100 条
seeds = generate_seeds(num_seeds=100)
```

**问题**：
- 句式模板化严重（"来点..."占 30%+）
- 类型分布不均（模糊需求 54%、直接请求 41%）
- 过度使用"推荐"一词

**尝试 2：分批并发 + 扰动策略**
```python
# 分 5 批并发，每批 30 条，添加随机扰动
batch_size = 30
num_batches = 5
# 每批不同的 temperature 和侧重方向
batch_variation = random.choice([
    "侧重直接请求（40%）",
    "侧重情绪驱动（30%）",
    "侧重倒装句（20%）",
    ...
])
```

**效果提升**：
- ✅ "其他"句式从 40% 提升到 57%
- ✅ "来点"句式从 30% 降至 21%
- ✅ 类型分布更自然

#### 扩展策略优化

**关键改进**：
1. **过采样去重**：生成 1.5 倍数据，去重后保留目标数量
2. **变体策略**：
   - 句式变化（60%）：倒装、省略、语气词
   - 场景泛化（20%）：不同驾驶场景
   - 口语噪声（20%）：填充词、重复、纠错
3. **批次并发**：4 个 worker 并发生成，提升效率

### 第三阶段：并发优化与边界样本生成

#### 边界样本自动生成

**问题**：13 个意图之间存在边界模糊情况，需要专门的边界样本

**解决方案**：
1. **自动识别混淆意图**：让 LLM 分析每个意图可能混淆的其他意图
2. **针对性生成边界样本**：针对每对混淆意图生成易混淆样本


**示例**：`music_recommendation` 的混淆意图
- `emergency_assistance`：用户可能通过电话联系紧急服务
- `transport_query`：部分交通场景中隐含拨号动作
- `poi_recommendation`：用户可能在获得推荐后立即说"打电话给这家餐厅"

#### 质量评估策略

**评估标准**：
1. 必须包含与意图相关的隐含词（如音乐意图需包含"听"、"歌"、"旋律"等）
2. 表达自然，像真人对语音助手说话
3. 长度 5-20 字

**评分机制**：0-5 分，保留≥3 分的数据

**评估模型**：Qwen3-1.7B（本地部署）

## 当前架构

### 完整 Pipeline 流程

```
┌─────────────────────────────────────────────────────────────┐
│                     数据生成 Pipeline                        │
├─────────────────────────────────────────────────────────────┤
│  步骤 1: 种子生成                                            │
│  - 使用并发策略（batch_size=30, 5 批）                        │
│  - 添加随机扰动（temperature、句式侧重）                      │
│  - 输出：intent_seeds.json                                   │
├─────────────────────────────────────────────────────────────┤
│  步骤 2: 种子扩展                                            │
│  - 对每个种子生成 8-10 个变体                                  │
│  - 过采样 1.5 倍 + 去重                                        │
│  - 输出：intent_expanded.json                                │
├─────────────────────────────────────────────────────────────┤
│  步骤 3: 边界样本生成                                        │
│  - 自动识别混淆意图                                          │
│  - 为每对混淆意图生成 10 条边界样本                           │
│  - 输出：intent_vs_confusing.json                            │
├─────────────────────────────────────────────────────────────┤
│  步骤 4: 质量过滤                                            │
│  - 使用 Qwen3-1.7B 评估每条数据                               │
│  - 0-5 分评分，保留≥3 分                                      │
│  - 输出：intent_keep.json / intent_discard.json              │
├─────────────────────────────────────────────────────────────┤
│  步骤 5: 数据合并                                            │
│  - 合并所有意图的保留数据                                     │
│  - 添加 intent 字段                                           │
│  - 输出：all_intents_dataset.json                            │
└─────────────────────────────────────────────────────────────┘
```

### 数据比例

```python
# PipelineConfig 配置
SEED_RATIO = 0.20      # 种子占比 20%
EXPAND_RATIO = 0.70    # 扩展占比 70%
BOUNDARY_RATIO = 0.10  # 边界占比 10%
ESTIMATED_PASS_RATE = 0.85  # 预估质量过滤通过率
```

**以目标 1000 条为例**：
- 需要生成总量：1000 / 0.85 ≈ 1176 条
- 种子数据：1176 × 0.20 ≈ 235 条
- 扩展数据：1176 × 0.70 ≈ 823 条
- 边界数据：1176 × 0.10 ≈ 118 条

## 快速开始

### 环境准备

```bash
# 安装依赖
pip install openai modelscope scikit-learn tqdm

# 配置 API（在 src/config/api_config.py 中）
QWEN_API_KEY = "your_api_key"
QWEN_API_URL = "https://dashscope.aliyuncs.com/compatible-mode/v1"
QWEN_MODEL_NAME = "qwen-plus"
```

### 生成数据

```bash
# 生成所有意图的数据（默认每个意图 1000 条）
python src/core/pipeline_manager.py --target 1000

# 生成指定意图的数据
python src/core/pipeline_manager.py --intent music_recommendation --target 1000

# 跳过质量评估（快速测试）
python src/core/pipeline_manager.py --target 100 --no-eval
```

### 评估意图分类效果

```bash
# 使用 Qwen3-1.7B 评估测试集
python src/evaluate_intent_classification.py
```

**输出**：
- `train_set.json` / `test_set.json`：训练集/测试集划分（8:2）
- `test_predictions.json`：测试集详细预测结果
- `test_metrics.txt`：各意图的 precision、recall、F1、FPR 指标

## 评估指标

### 意图分类评估

使用 Qwen3-1.7B 作为评估模型，对测试集进行意图预测：

**指标说明**：
- **Precision**：预测为该意图的样本中，实际为该意图的比例
- **Recall**：实际为该意图的样本中，被正确预测的比例
- **F1 Score**：Precision 和 Recall 的调和平均
- **FPR**：实际不为该意图的样本中，被错误预测为该意图的比例

**典型结果**（示例）：
```
music_recommendation:
  样本数：187
  Precision: 0.9245
  Recall:    0.8930
  F1 Score:  0.9085
  FPR:       0.0123
```

## 关键设计决策总结

### 为什么使用 LLM 而非规则？

✅ **LLM 优势**：
- 生成自然、口语化的表达
- 自动处理语言多样性
- 无需手动维护规则模板

❌ **规则劣势**：
- 规则有限，覆盖不全
- 随机组合不通顺
- 维护成本高

### 为什么分种子和扩展两阶段？

- **种子阶段**：保证质量，LLM 一次性生成高质量样本
- **扩展阶段**：保证数量，基于种子快速扩大规模
- **质量过滤**：双重保障，剔除不合格样本

### 为什么需要边界样本？

- 意图边界模糊是分类难点
- 专门生成易混淆样本，提升模型判别能力
- 自动识别混淆意图，减少人工标注

### 为什么使用并发策略？

- 单批次生成容易模板化
- 多批次 + 扰动提升多样性
- 线程池并发提升效率
